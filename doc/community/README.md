## How to Participate

1. **Use HEXA** -- Write programs, file bugs, share examples
2. **Contribute code** -- See [CONTRIBUTING.md](CONTRIBUTING.md) for setup and PR process
3. **Report issues** -- Use GitHub Issues with clear reproduction steps
4. **Propose features** -- Open a GitHub Discussion with your idea and its n=6 derivation
5. **Write documentation** -- The book and spec always need improvements
6. **Start a first issue** -- See [FIRST_ISSUES.md](FIRST_ISSUES.md) for beginner-friendly tasks

## Community Guidelines

### Be Respectful

Treat everyone with respect. We are a small community building something unusual. Disagreements about design are welcome; personal attacks are not.

### Be Mathematical

HEXA is not arbitrary. Every design decision traces back to n=6. When proposing changes, explain the mathematical basis. "It would be nice to have X" is weaker than "X maps to tau(6) because..."

### Be Precise

Code examples should be runnable. Bug reports should include reproduction steps. Feature proposals should include the n=6 derivation.

### Be Patient

This is a research language built by a small team. Response times may vary. If your PR sits for a while, a polite ping is fine.
